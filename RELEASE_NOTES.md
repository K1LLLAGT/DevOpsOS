# DevOpsOS 1.0 — Release Notes

DevOpsOS 1.0 is the first full release of a **developer operating system** designed for Android/Termux environments, combining:

- A modular WebView frontend
- A Node.js backend
- A full intelligence stack
- Commercial polish
- A subtle, integrated ARG

---

## Highlights

### Unified Intelligence (Arc C)

DevOpsOS 1.0 ships with a complete intelligence architecture:

- **Embeddings (C1)** — local embedding generator + vector store
- **Semantic Search (C2)** — hybrid ranking across files, commands, panels
- **Suggestions (C3)** — behavior‑driven smart actions and recency‑aware shortcuts
- **Insights (C4)** — correlation and rule‑based insight cards
- **Intelligence Bus (C5)** — event fabric connecting all intelligence modules

This turns DevOpsOS into a **coherent intelligence platform**, not just a collection of tools.

---

### Developer Power (Arc B)

- LSP integration for multiple languages
- Git panel for repo workflows
- Container panel for Docker/Podman
- Remote build runner over SSH
- Workspace indexer feeding semantic search

DevOpsOS becomes a **single cockpit** for local and remote development.

---

### Commercial Polish (Arc D)

- Local licensing engine
- Plugin marketplace with manifest validation
- Mock payment processor (safe, offline)
- Crash analytics and crashlog panel
- Branding metadata for name/version/channel

This makes DevOpsOS feel like a **real product**, ready for distribution.

---

### ARG Layer (Arc E)

A subtle hacker‑mystic ARG is woven through the system:

- Whispers, anomalies, clues, and an unlock phrase
- Final revelation: **“THE ROOT IS WITHIN”**
- Panels for clues and unlock state
- Designed to be atmospheric, not intrusive

---

### Verification System

DevOpsOS 1.0 includes a built‑in verification suite:

- Backend checks for arcs A–E, packaging, security, QA, release
- `/verify/run` endpoint
- Verification panel

Use it before tagging releases to ensure system integrity.

---

## Known Limitations

- LSP support depends on language servers being installed and reachable
- Remote build requires valid SSH configuration
- Marketplace uses a mock payment processor (no real payments)
- ARG is local‑only; no external services

---

## Upgrading

Future versions will maintain:

- Backward‑compatible plugin manifests where possible
- Stable intelligence bus event types
- Stable core routes for commercial and ARG layers

See `CHANGELOG.md` for detailed version history.
